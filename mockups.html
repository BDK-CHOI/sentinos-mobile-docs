서비스 5xx 오류율 급증 대응

HTTP 5xx 급증 — 앱/DB 오류 초동 · rule afqs5s6xaaghsc

CHECK 1

에러 로그에서 5xx 원인 확인

최근 5분 앱 로그에서 5xx/Exception 스택을 확인해 원인 유형(코드 오류 / DB / 외부연동)을 판별한다.

kubectl -n <ns> logs deploy/<app> --since=5m | grep -iE '5[0-9]{2}|exception'
CHECK 2

DB 커넥션풀·응답 확인

5xx가 DB 계열이면 커넥션풀 고갈/슬로우쿼리 여부를 본다.

curl -s localhost:8080/actuator/metrics/hikaricp.connections
CONTACT 3

원인이 DB면 DB운영파트 연락

커넥션 고갈·슬로우쿼리 등 DB 원인이면 업무·호스트·오류코드를 전달한다.

ACTION 4

일시 부하면 스케일아웃, 코드 결함이면 롤백

배포 직후 발생이면 직전 버전 롤백(담당 개발팀 확인 후).

kubectl -n <ns> rollout undo deploy/<app>
체크 = 콘솔에서 수행함
ACTION 5

해소 확인 후 Ack/조치내역 기록

5xx율이 임계 이하로 내려온 것을 확인하고 콘솔에 실제 조치를 기록한다.
