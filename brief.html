02 · NETWORK ARCHITECTURE

외부망 · DMZ · 내부망(폐쇄망) 구성

원칙: DMZ 데이터 미보관 · 내부 API 직접 노출 금지(BFF) · 로그 마스킹 서버사이드 · 푸시 페이로드 최소화 · MDM 단말 전제.

MDM 단말 · RN+WebView → HTTPS 443 → 외부 FW → DMZ WAF/Proxy → 내부 Next BFF → 관제·로그·SSO

ZONE A 외부망

  • MDM 관리 단말 (Android / iOS)
  • RN 셸 — WebView, 생체인증, 딥링크, (V1) 푸시 수신
  • 모바일 웹 UI — 브라우저 렌더 (실제 데이터는 BFF 경유)
  • 네이티브 스킴 — tel: / sms:
  • 스크린샷 방지 등 MDM 프로파일
  • 선택: per-app VPN / mTLS
  • ※ 내부 API 주소·토큰 원문 비노출

ZONE B DMZ

  • WAF — 위협 필터
  • 경량 리버스 프록시 (Nginx급)
    • TLS 종료
    • Rate Limit
    • 경로 화이트리스트
    • _next/static 캐시
  • 인증 경로 프록시 (방안 A 시 Keycloak OIDC 일부 노출)
  • 데이터 저장 금지 — 정적·중계만
  • 풀 API GW 제품은 소비자 증가 시 재검토

ZONE C 내부망 (폐쇄망)

  • Next.js SSR + BFF — 세션·집계·권한·마스킹
  • mobile-api (Spring) — InboxResponse 등 계약
  • Keycloak SSO/OTP (내부 정본)
  • sentinos-console API — 이벤트·Ack·owners·Runbook·knowledge 의미론 정본
  • 관제 Sentinos / Chronos / Biznos
  • 저장 PostgreSQL event_history · Loki/ES · VM/Prometheus
  • (V1) 푸시 발송 — 아웃바운드 FCM/APNs, 이슈 ID만

2.1 구간별 책임 한 줄

구간넣는 것넣지 않는 것
외부망 MDM 단말, RN 셸, WebView UI, 생체·딥링크·전화/SMS DB 접속, 내부 API 주소 하드코딩, 원문 로그 저장
DMZ WAF, TLS, Rate Limit, 경로 필터, 정적 캐시, (선택) OIDC 프록시 업무 데이터·로그 원문·세션 저장소·BFF 비즈니스 로직
내부망 SSR/BFF, mobile-api, Keycloak, 콘솔/관제/로그/메트릭/PG 외부 직접 노출 포트, 마스킹 전 데이터 클라이언트 전송
① DMZ 무데이터프록시·캐시만. 장애 원장·개인정보는 내부망.
② BFF 단일 입구브라우저/앱 → 내부 시스템 직접 호출 금지.
③ 서버사이드 마스킹권한·정책 적용 후 페이로드 전달. ▮▮▮ 표기.
④ 인증 이원화 검토A: OIDC 화이트리스트 / B: 대리 로그인 — 보안팀 협의.

03 · OUTCOMES

핵심 기능 및 효과 — 결론

Sentinos Mobile은 관제 화면의 축소판이 아니라, 야간·이동 중 3분 초동 대응을 위한 전용 도구다. 시안이 정한 위계(시간·활성 이슈·매뉴얼·연락)와 망 분리(외부·DMZ·내부)가 그 목적을 구조적으로 뒷받침한다.

3.1 핵심 기능

기능사용자 가치설계 포인트
활성 이슈 인박스 지금 손댈 일만 한 손에 Ack/Silence 제외 · 1분 폴링 · 경과 타이머
워룸형 상세 어디·무엇·얼마나 즉시 파악 로그/스냅샷 분기 · 담당 전체 · 액션 바
AI/Runbook 가이드 초동 매뉴얼 + 유사 이력 재사용 콘솔 진급조치·유사이슈·AI 정합 · 근거 없는 AI 비표시
Ack = Silence 상황인지 완료 + 알림 스누즈 3필수 입력 · 웹 Sentinos 의미론 동일
관계자 연락 이탈 없이 전화/SMS owners/resolve · 요약 자동 첨부
PC 핸드오프 (R10) 깊은 분석은 콘솔로 콘솔 링크 복사 · 읽기 전용 원칙 유지
상태 6종 · 신선도 “조용함” vs “고장” 혼동 방지 스테일 배너 · 서버 시각 오프셋

3.3 결론

  1. 시안이 곧 운영 언어다. 경과 타이머·활성 인박스·가이드 3단(매뉴얼·AI·유사 이력)은 새벽 한 손 조작에 맞춰져 있다.
  2. 망 분리가 보안과 속도를 동시에 만든다. 외부는 단말·셸, DMZ는 문지기, 내부는 데이터와 BFF — 역할이 겹치지 않는다.
  3. 콘솔과 경합하지 않는다. 콘솔은 전체 관리, 모바일은 지금 활성 이슈의 초동. 같은 데이터, 다른 자르기.
  4. 효과는 시간으로 측정한다. MTTA·3분 플로우·PC 부팅 0% — 관제 미학이 아니라 대응 시간 단축이 성공 기준이다.
