## 1. 한 줄 컨셉

> **시간이 주인공인 장애 대응 도구**  
> 대시보드를 줄인 것이 아니라, **경과·신선도·Silence**를 시그니처로 둔다.

사용자는 침대에서 한 손으로 조작하는 상황실 OP와, 새벽에 호출당한 개발/운영 담당자다.  
3분 안에 “지금 뭐가 터졌고, 어디고, 로그/그래프는, Ack 또는 누구에게 연락할지”만 남긴다.

## 2. 디자인 원칙

| # | 원칙 | 실천 |
|---|---|---|
| 1 | **대응 도구, 관제 축소판 아님** | 차트 과다·헬스 그리드 홈 금지. 인박스 = 활성 이슈만 |
| 2 | **시간이 주인공** | 경과 타이머를 카드·히어로에 최우선 위계로 배치 |
| 3 | **한 손·새벽·오조작 방지** | 터치 44px+, 하단 Ack/☎ 영역 분리, safe-area, 44px 미만 금지 |
| 4 | **이중 부호화** | 심각도는 항상 **색 + 텍스트 라벨**. 색맹·저대비 대비 |
| 5 | **조용함 ≠ 고장남** | 빈 인박스 vs 5xx/스테일/오프라인을 시각·카피로 분리 (R11) |
| 6 | **가려짐 2종 혼용 금지** | 무권한=블러+자물쇠 / 마스킹=`▮▮▮` |
| 7 | **데이터는 모노스페이스** | 시간·경과·호스트·오류코드·로그·수치 = JetBrains Mono |
| 8 | **읽기 전용 기본** | 명령 실행 UI 없음. 쓰기 = Ack · Runbook 체크(V1)뿐 |
| 9 | **콘솔과 팔레트 정합** | slate 스케일·도메인 의미는 sentinos-console 선례 |
| 10 | **모션 절제** | 시트 전환 + 타이머 틱만. `prefers-reduced-motion` 존중 |

## 3. 시각 언어

### 3.1 컬러 토큰 (임의 변경 절대 금지)

**다크가 기본.** `:root` = 다크, `.light` = 오버라이드. 첫 실행은 무조건 다크.

| 역할 | 다크 (`:root`) | 라이트 (`.light`) |
|---|---|---|
| bg | `#020617` | `#F3F5F7` |
| surface | `#0F172A` | `#FFFFFF` |
| surface-2 | `#1E293B` | `#E9EDF1` |
| text | `#F8FAFC` | `#17202A` |
| muted | `#94A3B8` | `#5C6874` |
| border | `rgba(255,255,255,.10)` | `#D9DFE6` |

**심각도 — 테마 블록 밖 고정** (의미색, 오버라이드 구조적으로 불가해야 함):

| 등급 | 색 |
|---|---|
| Critical | `#BF1B00` |
| Major | `#99440A` |
| Minor | `#D1CF2B` |
| 정상 / 해소 / 연락 액션 | `#3DD68C` (라이트 `#157F4C`) |

**액션 블루** — 버튼·링크·선택 상태에만. 정보 표시에 사용 금지.

| | 값 |
|---|---|
| 다크 | `#58A6E8` |
| 라이트 | `#1E6FBF` |

### 3.2 색 사용 규칙

- 심각도: 카드 **좌측 스트라이프 3.5px** · 뱃지 배경 위주.  
  **다크 배경 위 소형 텍스트에 심각도 색 단독 사용 금지** (대비 부족).
- 선택 칩·포커스 카드 외곽선: **액션 블루** (심각도 색과 경쟁 금지).
- System Health “GOOD”: 초록 아웃라인 필 등 **정상색** 계열.

### 3.3 타이포그래피

| 용도 | 폰트 | 비고 |
|---|---|---|
| UI / 본문 | **Pretendard** | 로컬 자가호스팅, 폴백 시스템 산세리프 |
| 데이터 | **JetBrains Mono** + `tabular-nums` | 시간·경과·오류코드·호스트·로그·수치 **전부** |

모노스페이스는 “관제의 재료”를 구분하는 시각 언어다.  
본문 산세리프와 섞어 쓰면 안 된다 — **재료는 mono, 설명은 Pretendard**.

### 3.4 시그니처 — 경과 타이머

| 위치 | 스펙 | 동작 |
|---|---|---|
| 인박스 카드 우상단 | 14px / 700 mono | “MM:SS 경과” (활성 이슈만 — Silence 뱃지 없음, v2.1) |
| 이슈 상세 히어로 | 34px / 800 mono | 실시간 틱 |
| 상세 · Ack 직후 (같은 화면 체류) | 동일 히어로 | **Silence 만료까지 카운트다운**으로 전환 |
| 가이드 앱바 (V1) | 경과 또는 Silence 카운트다운 | GD 스펙 |

모든 시간 표시는 **서버 시각 오프셋 보정** 전제 (R11②).

## 4. 레이아웃·반응형

### 4.1 넓이로 가른다 (기기/UA 아님)

| 뷰포트 | 모드 |
|---|---|
| ~767px | **모바일 단일 스택** — 목록 → 상세 → 로그/가이드가 페이지 전환 |
| 768px+ (`md`) | **태블릿 마스터·디테일** — 좌 인박스 **320px 고정** + 우 디테일 패널 |

- 회전·리사이즈 시 두 모드 **자연 전환** (같은 트리, CSS 분기).
- 태블릿 우측 = **스택 내비게이션** (상세 → 로그 → 가이드 push/pop, 브레드크럼/뒤로).
- 태블릿 하단 탭 바 = **좌측 마스터 패널 하단**. 우측은 자체 액션 바.

### 4.2 가로 치수 규칙

- 기본: `%` · `fr` · `flex` · `minmax()` · `clamp()`.
- **고정 px은 규격인 곳만** — 예: 인박스 패널 `320px`, 터치 타깃 `44px`. 사용 시 이유 한 줄.
- 세로는 고정값 허용.

### 4.3 내비게이션 모드 (폴더 = 답)

| 모드 | UI 하단 | 해당 화면 |
|---|---|---|
| **탭 내비게이션** | 탭 바 (이슈 · 이력 · 설정) | 탭 루트 |
| **스택 내비게이션** | 액션 바 (`Ack` \| `☎ 관계자`) | 상세 · 가이드 · 로그 상세 |

탭 바와 액션 바는 **같은 자리를 두고 배타 경쟁**.  
`usePathname()` 분기로 숨기지 말고, 라우트 그룹(tabs / stack)으로 표현한다.

### 4.4 WebView · 모바일 크롬

- safe-area: 하단 탭/액션/시트에 `env(safe-area-inset-bottom)`.
- 높이: `100dvh` ( `100vh` 금지 ).
- 문서 오버스크롤 바운스 억제. 시트 내부 스크롤 격리.
- `-webkit-tap-highlight-color: transparent`.
- 입력 `font-size >= 16px` (iOS 줌 방지).
- viewport `width=device-width` (300ms 탭 지연 제거).
- 관계자: 네이티브 `tel:` / `sms:`.

## 5. 상태 6종 — 모든 화면 필수

미구현 상태 화면은 **미완성**이다.

| 상태 | 시각·카피 방향 |
|---|---|
| **빈** | 인박스: “지금은 조용합니다 · 마지막 확인 HH:MM” |
| **로딩** | 스켈레톤 (레이아웃 점프 최소화) |
| **오류** | 원인 + 다음 행동 — “로그를 불러오지 못했습니다 · 다시 시도” |
| **권한 없음** | 블러 + 자물쇠 오버레이 / 액션 🔒 |
| **오프라인** | 네트워크 단절 명시 + 재시도 |
| **스테일** | 상단 배너 “HH:MM 이후 갱신 실패 — 표시 중인 목록은 과거입니다” + 목록 뮤트 |

**빈(200 조용함)** 과 **5xx/스테일(고장)** 을 같은 빈 상태로 합치지 않는다.

## 6. 정보 위계 — 화면별 컨셉

### 화면 1 — 이슈 인박스 (`/issues`)

**목표**: 0.5초 안에 “몇 건 · 가장 급한 것 · 얼마나 지났는지”.

- 헤더: “이슈” + **System Health** 뱃지(웹 산정값) + mono 캡션 “마지막 확인 · 1분마다 갱신”.
- 필터: 전체 / Critical / Major / Minor (액션 블루 선택).
- 카드:
  1행 — 심각도 라벨(mono) + **경과 타이머**  
  2행 — 오류 요약 (강조)  
  3행 — mono 메타 (시스템·업무·부서 · 담당 1인 · 발생 시각)
- 좌측 심각도 스트라이프.
- **Ack된 카드는 목록에 없음** (v2.1). Silence 뱃지 디자인은 인박스에 적용하지 않음.
- 새 이슈 배너 · 스크롤 점프 금지.

### 화면 2 — 이슈 상세 (`/issues/[id]`)

**목표**: 워룸형 한 장 스크롤 — “무엇·어디·얼마나·다음에 볼 재료”.

위계 위→아래:

1. 앱바 (‹ · 호스트 뱃지 · 새 이슈 미니 배지)  
2. **히어로** — 심각도·sourceType · 건수 · **대형 경과** · 제목 · mono 메타 2줄 · 담당 목록  
3. AI 요약 스트립 (V1, 근거 있을 때만)  
4. sourceType 본문 — 로그 리스트 **또는** 고정 스냅샷 그래프  
5. **하단 액션 바** — Ack | ☎  

히어로는 surface, 스트라이프 4px. 그래프는 SVG 스냅샷 1장, 조작 UI 없음.

### 화면 3 — 로그 상세

**목표**: 원문 가독 + 오류 하이라이트 + 시간 문맥.

- mono 8~12px, 줄바꿈 유지, Critical 틴트 하이라이트.
- 마스킹 `▮▮▮`, 스택 잘림 없음.
- 전후 로그 · 복사 · 공유(콘솔 링크).

### 화면 4 — Ack 바텀시트

**목표**: 필수 3필드 빠르게 · Silence 의미 명확.

- dim + 고정 높이 시트. 키보드 올라와도 저장 버튼 가림 금지.
- 조치예정 구간 mono 캡션. 2h 경고. 원인 퀵 칩. 재사용 출처 뱃지.
- 풀폭 액션 블루 CTA: “저장 — Ack 처리 · 알림 해제”.
- 토스트: “상황인지 완료 처리됨 · 알림 해제”.

### 화면 5 — 관계자 연락 레이어

**목표**: 이탈 없이 전화/SMS.

- Ack와 슬롯 공유. 행: 이니셜 · 이름/역할 · 전화 mono · [전화](초록) / [SMS](파랑).
- 캡션: SMS 요약 자동 첨부.

### 화면 6 — 가이드 (V1)

**목표**: Runbook 수행 + AI는 근거 있을 때만 · 저작 주체 톤 분리.

- caution 카드 → 확인 후 축약.
- Runbook 단계 카드(확인/명령/연락). 체크 1탭 / 해제 확인.
- AI 섹션은 surface-2로 톤 분리. 근거 미연결 항목 비렌더.
- 동일 액션 바 (수행 기록 첨부 캡션 · ☎ 미완료 뱃지).

### 화면 7 — 이력

**목표**: 재발 패턴 + 조치 재사용 코퍼스.

- 검색 · 필터 칩. 그룹 카드에 재발 횟수 mono/700.
- 원인 미확정 = 뮤트 · 재사용 버튼 없음.

### 화면 8 — 설정

**목표**: 신원·권한 확인 · 테마 · 버전. 잡동사니 금지.

- 카드 3 + 점선 예약(알림 V1).
- 테마 세그먼트: 다크(기본) | 라이트 | 시스템.

## 7. 인터랙션·카피

### 7.1 인터랙션

| 규칙 | 내용 |
|---|---|
| 터치 타깃 | ≥ 44px. Ack 영역과 ☎ 영역 분리 |
| 시트 | Ack / 관계자 동시 펼침 없음 |
| 그래프 | 팬/줌/탭 없음 |
| 명령어 | 실행 버튼 없음 · **[복사]** 만 |
| 모션 | 시트 + 타이머 틱 · reduced-motion |

### 7.2 카피 톤

- **원인 + 다음 행동** (“불러오지 못했습니다 · 다시 시도”).
- 성공은 결과 상태 언어 (“상황인지 완료 처리됨 · 알림 해제”).
- 빈 화면은 안심 + 신선도 (“지금은 조용합니다 · 마지막 확인 HH:MM”).
- 스테일은 과거 데이터임을 **명시**.

## 8. 컴포넌트 재사용 경계

구현 시 공통으로 분리할 단위 (프롬프트와 동일):

- 심각도 스트라이프 카드
